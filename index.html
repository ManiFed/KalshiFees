Architecture and methodology

How these estimates are built

Collector Architecture

The data collector is a Python API harvester that loads Kalshi fee multiplier history, enumerates live and historical event markets, fetches candlesticks, estimates fee revenue per candle, adds perpetual futures estimates, and writes daily and monthly CSV files. The static site reads the daily CSV directly in the browser and recomputes dashboard windows, totals, category mixes, cumulative totals, and chart series without a server.

API Inputs

The collector uses Kalshi's public trade API at https://external-api.kalshi.com/trade-api/v2. It calls /exchange/series-fee-changes for per-series taker multiplier history, /markets for open and active markets, /historical/markets for closed markets, live and historical candlestick endpoints for volume and price history, and /margin/markets for perpetual futures market notional estimates.

Event Fee Formula

For each candle, volume is multiplied by a rounded fee-per-contract estimate based on P × (1-P), where P is the candle mean price when available, then close price as fallback. Before May 1, 2025, the script treats fees as taker-only. From May 1, 2025 onward, it applies a taker fee plus a maker fee set to 25% of the taker multiplier, weighted by the configured maker/taker split.

Multiplier Logic

Series-specific fee multipliers come first from Kalshi's fee-change log. If no API change exists, the script falls back to standard event fees, reduced INX/NASDAQ100-style fees for known prefixes, and a zero-fee exclusion list for KXBTCY, KXCITRINI, and KXDOED. The fee era boundary is applied per candle, which matters for markets crossing the April-to-May 2025 change.

Candle Periods

Markets open for less than 24 hours are fetched with one-minute candles so short-lived sports or event markets are not flattened into a daily bucket. Multi-day markets use daily candles. Historical market requests include a six-hour buffer around open and close times to reduce missing edge activity.

Perpetual Futures

The collector treats perpetual futures separately from event contracts. It estimates revenue from notional volume using default maker and taker basis-point assumptions and excludes funding-rate transfers because those flow between traders rather than to Kalshi. Without daily perp history, the perp estimate is reported as an all-time adjustment and excluded from the daily chart.

Known Approximation

Candle mean price is a time-weighted midpoint proxy, not actual fill-price VWAP. Fee rounding is applied at market-candle granularity, not per individual fill, which can undercount small trades. Maker/taker split is assumed rather than observed. Missing or newly changed zero-fee/reduced-fee series can move the estimate. API gaps, rate limits, schema changes, and authenticated-only fee tiers can also affect completeness.

Dashboard Calculations

The page parses kalshi_fee_daily.csv, normalizes category columns prefixed with cat_, filters by first trade, last year, last six months, or last month, groups rows daily, weekly, or monthly, and recomputes total fees, average period fees, trailing annualized run rate, highest fee day, cumulative line data, and category totals for the visible window.
